PHASE I: BEGINNER - FOUNDATIONAL SQL (DDL, DML, and the Relational Model)
------------------------------------------------------------

**CORE CONCEPTS (Required for a "Noob"):**
1. THE RELATIONAL MODEL:
    * Definition of Relational Database Management System (RDBMS).
    * Understanding of Tables, Rows (Records/Tuples), and Columns (Attributes/Fields).
    * Keys: Primary Keys (unique identifier), Foreign Keys (establishing relationships between tables).
2. DATA DEFINITION LANGUAGE (DDL):
    * Syntax for CREATE TABLE: Defining column names and data types (INT, VARCHAR, DATE, etc.).
    * ALTER TABLE: Adding, modifying, or dropping columns.
    * DROP TABLE / TRUNCATE TABLE: Distinguishing between removing the entire table and just its data.
3. BASIC DATA MANIPULATION LANGUAGE (DML - CRUD):
    * SELECT: The fundamental query structure. Syntax: SELECT [columns] FROM [table].
    * INSERT: Adding new rows of data.
    * UPDATE: Modifying existing data based on a WHERE clause.
    * DELETE: Removing rows of data based on a WHERE clause.
4. FILTERING AND SORTING:
    * The WHERE Clause: Using Comparison Operators (=, <, >, !=).
    * Logical Operators: AND, OR, NOT for combining conditions.
    * Special Operators: LIKE (for pattern matching), IN (for lists), BETWEEN (for ranges), IS NULL.
    * ORDER BY: Sorting results (ASC/DESC).
    * LIMIT/FETCH: Restricting the number of rows returned.

PHASE II: INTERMEDIATE - ADVANCED QUERYING & DESIGN
------------------------------------------------------------

**CORE CONCEPTS (The "Adept" Toolkit):**
1. AGGREGATION AND GROUPING:
    * Aggregate Functions: COUNT(), SUM(), AVG(), MIN(), MAX().
    * The GROUP BY Clause: Grouping rows by common values to apply aggregate functions.
    * The HAVING Clause: Filtering results *after* grouping (WHERE cannot be used for aggregates).
2. SQL JOINS (The Foundation of Relational Data):
    * Understanding why Joins are necessary.
    * INNER JOIN: Only returns rows with matches in both tables.
    * LEFT/RIGHT JOIN (Outer Joins): Returns all rows from one table and the matched rows from the other (NULLs for unmatched).
    * FULL OUTER JOIN: Returns all rows when there is a match in either table.
    * SELF-JOIN: Joining a table to itself.
3. COMPLEX QUERY STRUCTURES:
    * Subqueries: Using a SELECT statement within another SQL query (as a column, as a list for WHERE, or as a table for FROM).
    * Common Table Expressions (CTEs): Using the WITH clause to create temporary, named result sets for better query readability and reuse.
    * UNION / UNION ALL: Combining the result sets of two or more SELECT statements (UNION removes duplicates).
4. DATA INTEGRITY:
    * Database Normalization: Formal rules (1NF, 2NF, 3NF) to design efficient, non-redundant schemas.

PHASE III: ADVANCED & UNIVERSITY-LEVEL SQL (Window Functions, Procedural SQL)
------------------------------------------------------------

**CORE CONCEPTS (The "Expert" Toolkit):**
1. WINDOW FUNCTIONS (The Analytic Powerhouse):
    * Definition: Functions that perform a calculation across a set of table rows that are somehow related to the current row, without reducing the number of rows returned (unlike GROUP BY).
    * Ranking Functions: ROW_NUMBER(), RANK(), DENSE_RANK().
    * Analytic Functions: LAG(), LEAD() (to access data from preceding/following rows).
    * Partitioning: The OVER (PARTITION BY...) clause.
2. PROCEDURAL SQL (T-SQL/PL/SQL):
    * Stored Procedures: Pre-compiled blocks of SQL code stored in the database.
    * Functions: Reusable code blocks that return a single value.
    * Triggers: Code that executes automatically in response to a DML event (INSERT, UPDATE, DELETE).
3. TRANSACTION MANAGEMENT (TCL):
    * COMMIT and ROLLBACK: Ensuring data changes are permanent or fully undone (ACID properties).
    * Setting Isolation Levels.
4. PERFORMANCE FUNDAMENTALS:
    * Understanding Indexes: How they speed up data retrieval and their trade-offs for writes.
    * Views: Stored queries for security and simplification.
